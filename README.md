# A-very-simple-companion-robot-for-emotion-detection-based-on-cv

**需要特别说明的是，创建本项目的目的是为了技术学习。** 它并非要做一个成熟、商业化的机器人，而是记录了一次利用普通笔记本和基础舵机，打通“感知-认知-运动”物理闭环的极客实验。主要验证了面部微表情识别、疲劳特征提取和大模型的情感干预逻辑。

## 核心架构

为了解决本地算力与物理运动之间的矛盾，项目采用了 C/S 分体架构：
- **上位机（算力与感官中枢）**：一台笔记本。负责运行 AI 模型（视觉、听觉、大语言模型），并接入 USB 摄像头、麦克风和音响。
- **下位机（机械执行躯干）**：Arduino 开发板 + 舵机云台。只负责执行上位机通过局域网或串口发来的指令，完成转头、视线追随等物理动作。

## 已实现的主要特性

- **视觉处理**：Python 直接推流，不再经过高延迟的外部虚拟摄像头中转。底层用 `MediaPipe` 解析面部 468 个 3D 关键点作为运动追踪的“靶心”；并行跑了 `Py-Feat` (ResMaskNet) 级联模型，实时提取 7 种微表情特征及置信度。
- **运动控制**：通过 `PID` 算法将视觉坐标系下的像素误差转化为舵机偏转角度，控制链路中串联了 `One Euro Filter`（一欧元自适应滤波器）来消除视觉识别抖动带来的机械震荡，实现了平滑的视线锁定。
- **认知核心**：基于本地部署的 `Qwen2.5-3B`（Ollama q4 量化）。通过手写的滑动窗口 FIFO 队列实现短期连续记忆管理；用动态 Prompt 注入技术，将视觉捕获的情绪状态作为大模型的“潜意识”输入，让其以心理学关怀的视角进行角色扮演。
- **语音表达**：通过 `Whisper` 模型实现语音转文字，用 `Edge-TTS`（或系统自带的 pyttsx3）完成语音合成。
- **并发反射机制**：采用生产者-消费者多线程模型。主线程维持 60 FPS 的视觉捕获与运动指令下发，后台线程以阻塞模式处理大模型推理和语音合成，确保机器人在“讲话”时，双眼仍能维持高频的目标追踪。

## 局限性与工程妥协

这是一个以学习为导向的实验原型，物理和算法层面都有不少粗糙的妥协，复刻前请知悉：

1. **心率检测失败**：最初设计的纯视觉 rPPG 技术（通过微血管颜色变化测心率）在实际部署时失败了。一旦机器人云台转动或环境光线变化，信号就会完全被噪声淹没。**目前代码里喂给大模型的生理信号，是用一段正弦波代码 (`bio_simulator.py`) 生成的模拟假数据**，仅用来验证大模型的触发逻辑。
2. **半手动的交互**：系统没有集成语音唤醒功能。所有对话需要用户**手动长按键盘的 `V` 键**来录音，体验上不够流畅。
3. **依赖线缆**：受算力限制，机器人无法脱离笔记本独立运行。机身必须拖拽 USB 线和音频线连接到旁边满载运行的电脑上。
4. **单目标锁定**：视觉逻辑被硬编码为只处理数组中的“第一张人脸”。如果画面中出现多个人，锁定逻辑会在不同目标间随机跳变，导致舵机云台输出混乱的指令。

## 快速开始

### 依赖安装

```bash
# 核心环境要求: Python 3.11
pip install "numpy<1.24" "opencv-python<4.10" "opencv-contrib-python<4.10"
pip install py-feat tensorflow requests sounddevice soundfile
```

*注：需提前在本地安装并启动 [Ollama](https://ollama.com/) 服务，并拉取 `qwen2.5:3b` 模型。*

### 启动流程

1. 检查 `config.yaml`，确认 Ollama 服务器的本地 IP 与端口。
2. 启动下位机通信模拟服务端（如果没有真实 Arduino 硬件）：
   ```bash
   python body_emulator.py
   ```
3. 启动主控制程序：
   ```bash
   python main.py
   ```
